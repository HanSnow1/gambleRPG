1. 프로젝트 개요
================================================================================

게임명: GambleRogue
장르  : HP 배팅 로그라이크 (Unity 2D + C#)
기간  : 2~3주 (MVP+)

핵심 규칙 (v3):
  - 런당 보스 1명 (풀 2종: 주사위 / 카드 중 랜덤)
  - 상점 대신 증강: 3장 제시 -> 1장 선택 (무료)
  - 증강 등급: 실버 / 골드 / 프리즘
  - 같은 증강 재선택 시 레벨업 (최대 Lv.3)
  - 유물: HP 최대치 증가, 공격력 증가만
  - 파산 방지 없음 (HP 0 = 즉시 게임오버)
  - 강화(장비) 시스템 없음

최소 시연 루프 (5~10분):
  보스 프리뷰 -> 일반 전투 2~3회 -> 증강 2회 이상 -> (이벤트) -> 보스 1전 -> 클리어/게임오버

3. 역할 A - 전투 & 보스 (Combat / Boss)
================================================================================

담당: 싸움의 재미 + 보스 1명 전투 + 보스 데이터

[담당 폴더]
  Assets/Scenes/Scripts/Combat/     (주 담당)
  Assets/Scenes/Scripts/Data/       (BossDefinition만)
  Assets/Scenes/Scripts/UI/         (BossPreviewUI, CombatUI 유지)
  Assets/Scenes/ScriptableObjects/Bosses/

[할 일 목록]
  우선순위 1  전투 마무리 (Play 테스트, UI 위치, 버튼 글자 Safe/Risk/All-in)
  우선순위 2  BossDefinition ScriptableObject (주사위/카드 2종 에셋)
  우선순위 3  RunPlanner - 런 시작 시 보스 1명 랜덤 (C와 협의 가능)
  우선순위 4  보스 프리뷰 UI (이름, 규칙, 약점, 패턴 텍스트)
  우선순위 5  보스전: 주사위/카드 규칙 반영 (BossGambleResolvers.cs)
  우선순위 6  전투 수치표 (3버튼 확률/배당/실패 페널티)

[산출물]
  - Data/BossDefinition.cs
  - Combat/BossGambleResolvers.cs
  - UI/BossPreviewUI.cs
  - ScriptableObjects/Bosses/ Boss_Dice.asset, Boss_Card.asset

[다른 팀원에게 제공]
  - BossDefinition 필드 확정 (이름, GambleType, HP, 패턴 텍스트)
  - 전투 modifier 훅: GetSuccessChance(float baseChance) 등 (B가 증강 연동)

[1일차 목표]
  Play 되는 전투 확인 + Boss_Dice / Boss_Card 에셋 2개 생성

4. 역할 B - 증강 (Augment)

담당: TFT식 증강 (3택1, 등급, 레벨업)

[담당 폴더]

  Assets/Scenes/Scripts/Run/        (AugmentSelector, PlayerAugmentState)
  Assets/Scenes/Scripts/UI/         (AugmentPickUI)
  Assets/Scenes/ScriptableObjects/Augments/

[할 일 목록]
  우선순위 1  AugmentDefinition + 등급 enum (Silver/Gold/Prismatic)
  우선순위 2  증강 12~15종 에셋 + 수치표 (등급 x Lv1~3)
  우선순위 3  AugmentSelector - 3장 뽑기, 보스 타입 가중치 +15%
  우선순위 4  같은 증강 재선택 = 레벨업 (최대 Lv.3)
  우선순위 5  증강 선택 UI (3카드, 등급 색, Lv 미리보기)
  우선순위 6  A의 전투에 확률/피해 modifier 연결

[산출물]
  - Data/AugmentDefinition.cs
  - Run/AugmentSelector.cs, Run/PlayerAugmentState.cs
  - UI/AugmentPickUI.cs
  - Combat/AugmentCombatModifiers.cs

[다른 팀원과 약속]
  - HP/공격력은 증강에 넣지 않음 (유물 = C 담당)
  - RunState의 증강 리스트는 C 소유, B는 API로 읽기/쓰기만

[1일차 목표]
  AugmentDefinition.cs + 증강 3개 에셋 + 버튼 1개로 3택1 UI 테스트

8. Git 규칙 (충돌 방지)
================================================================================

  1. 동시에 Main.unity 수정 금지 - C만 맵/씬 통합
  2. 브랜치 예시:
       feature/combat-boss   (A)
       feature/augment       (B)
       feature/run-map       (C)
  3. 공용 스크립트 GameFlowController / RunState 수정 시 팀 채팅 공지
  4. ScriptableObject는 각자 폴더에만 추가:
       Bosses/  Augments/  Relics/
  5. 매주 금요일 main 브랜치에 merge
